# Agent Pack Maintenance Checklist

Use this checklist when adding or maintaining agent packs, templates, checklists, or navigation references.

## Agent File Fit

- [ ] Agent file names are clear, lowercase, hyphenated, and role-specific.
- [ ] Directory placement matches the durable domain responsibility.
- [ ] No duplicate or overlapping domain placement is introduced.
- [ ] No placeholder-only agents or directories are added.

## Required Agent Structure

- [ ] Purpose is clear.
- [ ] Responsibilities are specific and bounded.
- [ ] Non-Goals are explicit.
- [ ] Review / Check Criteria are actionable.
- [ ] Output Expectations are clear.
- [ ] Public OSS Hygiene Reminder is present.

## Documentation Maintenance

- [ ] Cross-references are useful, minimal, and current.
- [ ] Taxonomy update is included when actual agent coverage changes.
- [ ] Roadmap update is included when scope moves between future and current coverage.
- [ ] Tool-routing update is included where routing guidance changes.
- [ ] Future items are not marked complete prematurely.
- [ ] Templates and checklists are presented as operating artifacts, not agents.

## Scope Control

- [ ] No broad rewrites are included without explicit scope.
- [ ] Existing files were not moved or renamed without owner approval.
- [ ] Documentation-only boundary is preserved.
- [ ] No application code, implementation code, scripts, automation, workflows, package files, lockfiles, dependencies, generated files, screenshots, diagrams, assets, releases, tags, deployments, or artifacts were added.

## Public OSS Hygiene

- [ ] No private AI session links, task URLs, local paths, generated-by signatures, private tool metadata, private prompts, model traces, tool logs, screenshots, secrets, credentials, customer data, private dashboards, private roadmap details, private strategy, legal analysis, compliance analysis, or internal-only notes are present.
