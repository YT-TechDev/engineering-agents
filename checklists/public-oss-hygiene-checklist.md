# Public OSS Hygiene Checklist

Use this checklist before publishing repository-facing docs, PR bodies, commits, comments, templates, or checklists.

## Forbidden Public Repository Content

- [ ] ChatGPT shared links.
- [ ] Codex task URLs.
- [ ] Claude session URLs.
- [ ] Private AI session links.
- [ ] Local paths.
- [ ] Generated-by signatures.
- [ ] Private tool metadata.
- [ ] Internal-only notes.
- [ ] Secrets, credentials, tokens, API keys, private keys, cookies, connection strings, or signing material.
- [ ] Private prompts, hidden prompts, production prompts, system prompt fragments, or developer prompt fragments from private tools.
- [ ] Model traces, tool logs, AI transcripts, private logs, stack traces, crash reports, dashboard exports, or analytics exports.
- [ ] Private dashboards, analytics URLs, deployment URLs, preview URLs, environment URLs, provider dashboards, or repository settings screenshots.
- [ ] Customer data, personal data, participant data, user content, support tickets, transcripts, recordings, survey responses, or sensitive data.
- [ ] Private roadmap, strategy, pricing, revenue, vendor, procurement, contract, legal, compliance, or security questionnaire details.
- [ ] Screenshots containing private data.

## Safe Alternative

- [ ] Summarize AI-assisted work, decisions, review findings, and validation in plain public-safe text.
- [ ] Name commands or checks without pasting private logs or tool traces.
- [ ] Describe owner decisions without exposing private links, dashboards, customer data, procurement notes, legal analysis, or internal-only context.
