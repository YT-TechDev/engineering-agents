# Credit-Saving Practices

Use focused workflows to reduce unnecessary AI and automation usage.

## Recommended Practices

- Read only the files needed for the current task before expanding scope.
- Write scoped prompts with clear files to change, non-goals, and validation steps.
- Prefer small PR-sized documentation edits over broad rewrites.
- Reuse templates instead of starting from scratch.
- Avoid running builds or dependency installation for docs-only changes.
- Validate structure and Markdown formatting with targeted checks.

## For Reviews

- Start with the diff and relevant instructions.
- Check public OSS hygiene before deeper wording review.
- Ask for targeted follow-up changes instead of reworking unrelated sections.
